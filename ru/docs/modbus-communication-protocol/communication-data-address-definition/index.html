      8.2.2 Данные нефункциональных кодов

      Данные нефункциональных кодов могут быть разделены на функциональные и нефункциональные данные. Последние включают операционные команды, операционное состояние, операционные параметры, информацию об авариях и т. д.

      8.2.2.1 Данные функциональных кодов

      Данные функциональных кодов являются важными параметрами настройки инвертора, разделенными на функциональные параметры группы P и группы A. Группы параметров представлены следующим образом:

      Данные функциональных кодов Диапазон данных Атрибутный код
      Группа P P0, P1, P2, P3, P4, P5, P6, P7, P8, P9, PA, PB, PC, PD, PE, PF Читаемые и записываемые
      Группа A A0, A1, A2, A3, A4, A5, A6, A7, A8, A9, AA, AB, AC, AD, AE, AF Читаемые и записываемые

      Определение адресов коммуникации для данных функциональных кодов выглядит следующим образом:

      1. При чтении данных функциональных кодов для группы P0~PF и группы A0~AF, старшие шестнадцать бит адреса общения непосредственно представляют номер функциональной группы, а младшие шестнадцать бит представляют порядковый номер функционального кода в функциональной группе. Например:

        • Функциональный параметр P0-16: его адрес общения - F010H, где F0H представляет функциональный параметр группы P0, а 10H представляет шестнадцатеричный формат данных порядкового номера 16.
        • Функциональный параметр AC-08: его адрес общения - AC08, где ACH представляет функциональный параметр группы AC, а 08H представляет шестнадцатеричный формат данных порядкового номера 08.

      2. При записи данных функциональных кодов для группы P0~PF функциональных кодов старшие шестнадцать бит адреса общения различаются между 00~0F или F0~FF в зависимости от того, требуется ли запись в EEPROM, а младшие шестнадцать бит непосредственно представляют порядковый номер функционального кода в функциональной группе. Например:

        • Запись функции для параметра P0-16:
          • Если требуется запись в EEPROM, его адрес общения - F010H.
          • Если запись в EEPROM не требуется, его адрес общения - 0010H.

      3. При записи данных в EEPROM для группы A0~AF функциональных кодов старшие шестнадцать бит адреса общения различаются между 10~4F или A0~AF, а младшие шестнадцать бит непосредственно представляют порядковый номер функционального кода в функциональной группе. Например:

        • Запись функции параметра AC-08:
          • Если требуется запись в EEPROM, его адрес общения - AC08H.
          • Если запись в EEPROM не требуется, его адрес общения - 4C08H.

      8.2.2.2 Данные нефункциональных кодов

      Данные нефункциональных кодов Диапазон данных Атрибутный код
      Данные состояния Мониторинг параметров группы U, описание неисправностей инвертора, операционное состояние инвертора Читаемые
      Управляющие параметры Управляющие команды, параметры связи, управление цифровыми выходными терминалами, управление аналоговым выходом AO1, управление аналоговым выходом AO2, управление выходом высокоскоростного импульса (FMP), параметр инициализации Записываемые

      8.2.3 Данные состояния

      Данные состояния включают мониторинговые параметры группы U, описание неисправностей инвертора и операционное состояние инвертора.

      1. Мониторинговые параметры группы U
        Описание мониторинговых данных группы U можно найти в соответствующем руководстве по функциональным параметрам группы U0. Определение адреса выглядит следующим образом:
        U0~UF, где старшие шестнадцать бит адреса общения изменяются от 70 до 7F, а младшие шестнадцать бит представляют собой порядковый номер мониторящего параметра в группе. Например, U0-11 имеет адрес общения 700BH.

      2. Описание неисправностей инвертора
        При общении для чтения неисправности инвертора адрес фиксируется на 8000H. Чтением данных по этому адресу верхний компьютер может получить текущий код неисправности инвертора. Описания кодов неисправностей можно найти в Главе Пять, Функциональный код P9-14.

      3. Операционное состояние инвертора
        При общении для чтения операционного состояния инвертора адрес фиксируется на 3000H. Чтением данных по этому адресу верхний компьютер может получить текущую информацию об операционном состоянии инвертора, определенную следующим образом:

        Адрес общения для операционного состояния инвертора Читаемый статус
        3000H 1: Вращение вперед
        2: Вращение назад
        3: Остановлен

      8.2.4 Управляющие параметры

      Управляющие параметры включают управляющие команды, управление цифровыми выходными терминалами, управление аналоговым выходом AO1, управление аналоговым выходом AO2 и управление высокоскоростным импульсным выводом FMP.

      8.2.5 Управляющие команды

      При выборе P0-02 (источник команды) равным 2: управление посредством связи, верхний компьютер может управлять запуском, остановкой и другими связанными командами инвертора через этот адрес общения. Определения управляющих команд следующие:

      Адрес общения для управляющих команд Содержимое команды
      2000H 1: Пуск вперед
      2: Пуск назад
      3: Пуск ползания вперед
      4: Пуск ползания назад
      5: Останов свободного хода
      6: Остановка с торможением
      7: Сброс ошибки

      8.2.6 Управление цифровыми выходными терминалами

      При выборе функции цифрового выходного терминала равной 20: управление посредством связи, верхний компьютер может контролировать цифровые выходные терминалы инвертора через этот адрес общения. Определения следующие:

      Адрес общения для управления цифровыми выходными терминалами Содержимое команды
      2001H BiT0: Управление выходом DO1
      BiT1: Управление выходом DO2
      BiT2: Управление выходом RELAY1
      BiT3: Управление выходом RELAY2
      BiT4: Управление выходом FMR
      BiT5: Управление выходом VDO1
      BiT6: Управление выходом VDO2
      BiT7: Управление выходом VDO3
      BiT8: Управление выходом VDO4
      BiT9: Управление выходом VDO5

      8.2.7 Управление цифровыми выходными терминалами

      Когда функция цифрового выходного терминала выбрана как 20: управление коммуникацией, верхний компьютер может управлять цифровыми выходными терминалами инвертора через этот коммуникационный адрес. Определения следующие:

      Коммуникационный адрес для управления цифровыми выходными терминалами Содержание команды
      2001H BiT0: Управление выходом DO1
      BiT1: Управление выходом DO2
      BiT2: Управление выходом RELAY1
      BiT3: Управление выходом RELAY2
      BiT4: Управление выходом FMR
      BiT5: VDO1
      BiT6: VDO2
      BiT7: VDO3
      BiT8: VDO4
      BiT9: VDO5

      8.2.8 Управление аналоговым выходом AO1, AO2, и выходом высокоскоростного импульса FMP

      Когда функции аналогового выхода AO1, AO2, и выхода высокоскоростного импульса FMP установлены как 12: настройка коммуникации, верхний компьютер может управлять аналоговым выходом и выходом высокоскоростного импульса инвертора через этот коммуникационный адрес. Определения следующие:

      Выходной терминал Коммуникационный адрес для управления выходом Содержание команды
      AO1 2002H 0～7FFF представляет 0%～100%
      AO2 2003H 0～7FFF представляет 0%～100%
      FMP 2004H 0～7FFF представляет 0%～100%

      8.2.9 Инициализация параметров

      Когда необходимо выполнить операции инициализации параметров на инверторе через верхний компьютер, используется эта функция. Если PP-00 (пользовательский пароль) не равен 0, сначала необходимо выполнить проверку пароля. После успешной проверки операцию инициализации параметров можно выполнить верхним компьютером через 30 секунд.

      Коммуникационный адрес для проверки пользовательского пароля - 1F00H. Прямо на этот адрес следует записать правильный пользовательский пароль для завершения проверки пароля.

      Адрес для инициализации параметров - 1F01H, и его содержание данных определено следующим образом:

      Коммуникационный адрес для инициализации параметров Функция команды
      1F01H 1: Восстановить заводские параметры
      2: Очистить информацию о записях
      4: Восстановить пользовательские резервные параметры
      501: Резервное копирование текущих пользовательских параметров
